Interview Cheat Sheet

Click any question — these are the exact things interviewers ask about this project.

"What is Sentiment Analysis?" ▶
Say this: "Sentiment Analysis is an NLP task that identifies and classifies the emotional tone of text — typically as positive, negative, or neutral. It's widely used for analyzing customer reviews, monitoring social media, and measuring brand reputation." What is NLP? Natural Language Processing — a branch of AI that deals with understanding and generating human language using machines.
"Explain TextBlob and Naive Bayes" ▶
TextBlob is a Python NLP library that uses a Naive Bayes classifier trained on IMDb movie reviews to predict sentiment. Naive Bayes is a probabilistic ML algorithm based on Bayes Theorem. It calculates the probability a text belongs to a class (positive/negative) based on the words it contains. The "naive" part means it assumes all words are independent of each other — which isn't true in real language, but it still works very well in practice. It returns: polarity (−1 to +1) and subjectivity (0 = fact, 1 = opinion).
"What is VADER and why use two models?" ▶
VADER (Valence Aware Dictionary and sEntiment Reasoner) is a rule-based analyzer, not ML. It uses a hand-crafted dictionary of ~7,500 words with sentiment scores. VADER understands things TextBlob misses: GREAT!!! scores higher than great because of caps and punctuation. It's specially designed for social media and short text. Why two models? This is called an ensemble approach. Combining multiple models reduces the weaknesses of any single one and produces more reliable results. I average both compound scores for the final prediction.
"Walk me through your code architecture" ▶
Say this: "The app follows a standard client-server architecture." 1. Frontend (HTML/CSS/JS) — the user types text, JavaScript sends a POST /analyze request with a JSON body. 2. Flask backend (Python) — receives the request, validates the input, runs the NLP pipeline, and returns a JSON response. 3. NLP pipeline — text is cleaned with regex (preprocessing), passed to TextBlob and VADER, scores are averaged (ensemble), and token-level word scores are computed. 4. Frontend renders — JavaScript unpacks the JSON and updates the UI with bars, chips, and labels.
"What are the limitations? How would you improve it?" ▶
Answering this shows maturity. Say: • Sarcasm: "Oh great, another Monday" scores positive — both models miss sarcasm entirely. • Context: "This film is sick" — "sick" can mean good in slang but scores negative. • Language: Only works well in English. How to improve: "I would replace TextBlob with a fine-tuned BERT or RoBERTa model from HuggingFace. These transformer models understand full sentence context, not just individual words, so they handle sarcasm and slang much better."
"Why Flask? What is a REST API?" ▶
Flask is a lightweight Python web framework. I chose it because it's simple, widely used for ML model serving, and fast to build with. REST API stands for Representational State Transfer. It's a standard way for a frontend and backend to communicate over HTTP using JSON. My app has three endpoints: GET / returns the HTML page, POST /analyze receives text and returns analysis, GET /examples returns sample sentences. In production, engineers often use FastAPI instead because it's faster and has automatic documentation — that's my next step to learn.
